Sinatra

Week 3 Review

Posted by chrisharry on September 18, 2015

Web Applications

This week has been my toughest week so far here at makers. Probably due to the lack of sleep over the last couple of days but also being introduced to Sinatra. Our projects for this week was to create a web application using our Battleship ruby code from the week before and transferring it to the web using Sinatra.

Sinatra is a web application library and domain-specific language written in Ruby. Its main purpose is the quick and easy building of web applications. Although tough to start with once you understand the way your supposed to link your application logic to your controller and views things become a lot easier. In your controller you write your routes, this is a HTTP method paired with a URL-matching pattern. Each route then has a block of code which can link you to a view (what the user will see) and can also use your application code (Battleships).


get '/home_page' do
  #show something
  if @name
    erb :home_page
  else
    erb :ask_name
  end
end

post '/home_page' do
  #create something
  @name = params[name]
  redirect ('/home_page')
end

The code above shows that when you go to the url '/home_page' it will show you the html based on the home_page.erb file in your view. if @name exists it may show something like 'Hi @name'. If it doesn't exist it will send you to the ask_name page where a form may ask you to input your name. This form can have action: '/home_page' with the method = post which will then send you to the post 'home_page' do above and save @name to the value you just entered into the form. This will then redirect you to the '/home_page'nwhere the @name now exists and so will show you the :home_page which might say 'Hi Chris'.

Learning to link all these parts together was a slow process but by having an actual visual representation of your work in the form of a website made it easier to slog through. By the end of the week I now have a working game, albeit just playing against the computer, but I didn't really see the point in making a two player game of battleship on one screen. The only frustrating thing is the lack of javascript so far, we've gone from 70;s command line based code to now 90's looking websites, so it'l be nice to be able to start using things like JQuery to spice it up.

I believe the weekend challenge will be to create another web application of a small game but I'm not sure yet. Hopefully it won't be anything too substantial like last weekend but I know I'll be spending a lot of time on whatever it is to get it to the standard I want. It'll just be a toss up between pretty code or watching the opening weekend of the Rugby world cup. Pretty sure I know which of those will end up winning.
