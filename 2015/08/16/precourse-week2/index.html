Pre-Course Week 2

Moving along nicely

Posted by chrisharry on August 16, 2015

Round Up

Second week into the Pre-Course and things are running smoothly. After the pains I went through setting up all the software (on windows) last week, this week has been a breeze. Having completed the Git / Github work for week 2 already, I moved on to finishing off Chris Pine's Learn to Program. I've completed up till chapter 12 and so only two more to go! With the exception of chapter 10, on recursion, things haven't been too confusing. Recursion was a rather strange concept to wrap my head around at first, but after completing the exercises and playing around you just learn by seeing what does and doesn't work.

Recursion is a technique where you create a method that calls itself. I'll try explain through an example I definitely didn't steal off stackoverflow:

How do you empty a vase containing N flowers?

Answer: if the vase is not empty, you take out one flower and then you empty a vase containing N-1 flowers.

This code checks first if the number of flowers in a vase is greater than 0. If that is false then the program will simply return 'The vase is empty' But if it is true it'll call the method 'empty' on the (numberFlowersInVase - 1). This will keep going and going until the vase is empty.

I know your asking the same thing I did, "Why can't you just use a loop?" and to be honest you could, but this is a lot more 'elegant'. Now I don't know about elegant but it is a lot tidier... and cooler.

On Friday, week 2's challenge was released, the Octocat challenge. This involved the basic uses of Git and Github commands and involved the creation of a repository containing files which themselves contained questions. The task was to then create a separate branch of this repository which would be made up of the same files, but instead of the files containing questions they would contain the answers. Essentially creating two versions of the files, one with questions, one with answers. We also had to fork a provided repository, add our adaptations to it and then create a pull request. This is just editing a file on someone else's repository and then submitting your changes for them to review. All very easy but absolutely essential things to know and understand.

Only 2 weeks to go now until the real work begins and I can't wait.

Chris
